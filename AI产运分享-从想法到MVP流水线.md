## 📍 Step 1: 需求分析 — 头脑风暴工作坊

### 讲解要点

**开场白参考：**
```
大家平时有没有这种感觉？
- 开会2小时，结果"没结论"
- 需求评审反复改，改完开发说"这不是我理解的"
- 产品说"我要做A"，开发做出来是"B"

今天我教大家一套方法：
用AI帮你做需求分析，从"凭感觉"变成"有框架"
```

### 核心方法

```
业务背景 → AI结构化拆解 → 需求优先级 → 技术评估
```

### 工具

- **OpenClaw brainstorming skill** — Socratic追问拆解
- 或直接用 **ChatGPT / Claude**

### 演示Prompt模板（现场用）

```
我是一个[公司类型]的产品经理
我们现在遇到了[问题描述]
用户群体是[用户画像]
目标[业务指标提升]

请帮我分析：
1. 这个问题背后真正的原因是什么？
2. 用户有哪些未被满足的需求？
3. 有哪些可能的解决方案？
4. 你会建议先做哪个？为什么？
```

### 产出物

- ✅ 结构化需求文档
- ✅ 用户故事
- ✅ 优先级初步判断

## 📍 Step 2: 流程图+架构图（合并，5分钟）

### 讲解要点

**核心观点：**
```
传统方式：Visio/ProcessOn手画 → 1小时
AI方式：描述需求 → 自动生成 → 5分钟

为什么产运要会画流程图和架构图？
1. 和开发沟通不在一个频道？ → 画出来
2. 不知道技术边界？ → 画出来就知道了
3. 不理解系统依赖？ → 画出来就清晰了
```

### 工具

| 工具 | 地址 | 适合场景 |
|------|------|---------|
| **ProcessOn AI** | processon.com | 流程图，国产首选 |
| **Whimsical AI** | whimsical.com | 流程图+架构图，英文 |

### Prompt模板（二选一演示）

**流程图Prompt：**
```
帮我画一个[功能名称]的业务流程图

包含：
- 主流程（用户主要操作路径）
- 异常流程（出错/边界情况）
- 决策节点（需要判断的地方）

用户角色：[用户/运营/管理员]
```

**架构图Prompt：**
```
根据以下需求，帮我生成产品架构图

包含：
- 前端需要哪些模块/页面
- 后端需要哪些服务
- 需要哪些数据存储
- 需要对接哪些第三方服务

格式：Mermaid代码 或 直接生成图片
```

### 产出物

- ✅ 泳道图
- ✅ 模块划分 + 服务关系图

### 讲解建议

```
只选一个工具演示，推荐ProcessOn AI（中文界面）

流程：
1. 打开ProcessOn AI
2. 输入Prompt
3. 5秒生成结果
4. 展示"这就是AI画出来的"

重点：不需要手画，描述需求AI自动生成
```

## 📍 Step 3: Prompt工程

### 讲解要点

```
AI时代最重要的技能是什么？
不是"写代码"，是"写指令"

同样的工具，为什么有人用得好，有人用不好？
差距就在Prompt

核心公式：
Prompt = 角色 + 任务 + 约束 + 成功标准
```

### 工具

- **ChatGPT** / **Claude**（国内可直接用）

### 核心公式

```
┌─────────────────────────────────────────────────────┐
│  Prompt = 角色 + 任务 + 约束 + 成功标准              │
└─────────────────────────────────────────────────────┘
```

### 示例Prompt模板

#### PRD生成

```
你是一个资深产品经理

角色：你正在为一个[产品类型]设计[功能名称]
任务：帮我写一份完整的PRD文档

约束条件：
- 目标用户是[用户群体]
- 需要包含[具体功能点]
- 限制条件是[技术/资源限制]
- 文档字数控制在2000字以内

成功标准：
- 能够让开发团队准确理解需求
- 包含用户故事、验收标准、非功能需求
```

#### 用户画像

```
你是一个用户研究专家

任务：为[产品/功能]生成3个典型用户画像

请包含：
- 用户基本信息（年龄、职业、技术背景）
- 用户目标（想达成什么）
- 用户痛点（现在的问题）
- 使用场景（在什么情况下会用）

格式：用表格+简短描述
```

### 产出物

- ✅ 可执行的AI指令，粘贴进Stitch

## 📍 Step 4: 原型设计 — Google Stitch

### 讲解要点

```
传统流程：
需求 → 画原型 → 评审 → 修改 → 再评审
         ↓
        3-5天

AI流程：
需求 → AI生成UI → 评审 → 微调
         ↓
        30分钟出初稿

核心逻辑：
你描述清楚"我要什么"，AI帮你画出来
```

### 工具

**地址：** https://stitch.withgoogle.com/

### Stitch使用方法

**基础Prompt公式：**
```
为[产品类型]设计一个[页面类型]

包含以下元素：
1. [元素1，如导航栏]
2. [元素2，如用户头像]
3. [元素3，如内容列表]

设计风格：[简洁/现代/专业/活泼]
目标用户：[用户群体]
核心目标：[用户进来要完成什么任务]
```

### 现场演示Prompt示例

#### 示例1: 电商首页

```
Design a mobile shopping app homepage for young women who love fashion.
Include: hero banner, category tabs, product grid, shopping cart icon, user profile.
Style: Modern, clean, pink and white color scheme.
Goal: Help users discover and purchase trending fashion items quickly.
```

#### 示例2: 内部审批流

```
Design an internal approval workflow interface for managers.
Include: pending approval list, approval/reject buttons, detail panel, history timeline.
Style: Professional, enterprise, blue color scheme.
Goal: Let managers quickly review and make decisions on team requests.
```

#### 示例3: 数据看板

```
Design a data dashboard for operations team to monitor daily KPIs.
Include: date range selector, KPI cards, trend charts, export button, alert panel.
Style: Data-focused, clean, dark mode optional.
Goal: Help ops team quickly identify anomalies and take action.
```

### 产出物

- ✅ 高保真设计稿（分钟级）

## 📍 Step 5: MVP开发 — Google Antigravity

### 讲解要点

```
原型画完了，下一步是什么？
传统：找开发排期 → 等2周 → 看效果

现在：把设计给AI → 30分钟出可运行版本

Antigravity的核心能力：
1. Agent Manager — 调度多个AI帮你干活
2. Editor View — AI辅助写代码
3. 自动测试 — AI跑完告诉你对不对
4. Artifacts — 生成截图/视频让你审核
```

### 工具

**地址：** https://antigravity.google/

### 核心概念解释

| 概念 | 通俗解释 |
|------|---------|
| **Agent** | AI助手，但能自主行动（不只是回答问题） |
| **Manager View** | 指挥台，可以看到多个AI同时工作 |
| **Artifacts** | AI的工作成果汇报（截图/视频），方便你审核 |
| **Editor View** | 传统编程界面，AI在旁边辅助你写 |

### Antigravity工作流

```
Step 1: 描述需求
        ↓
Step 2: Agent理解 → 规划 → 执行
        ↓
Step 3: 生成Artifact汇报进展
        ↓
Step 4: 你审核，给反馈
        ↓
Step 5: Agent修正，继续执行
```

### MVP场景示例

```
创建一个用户注册页面，包含：
- 邮箱/手机号输入
- 密码设置（需包含大小写+数字）
- 用户协议勾选
- 注册按钮

注册成功后跳转到欢迎页

要求：
- 响应式设计（手机/电脑都正常）
- 输入实时校验
- 密码强度提示
```

- ✅ 可运行的MVP版本

## 💡 三个核心观点

```
1. AI不是替代你，是放大你
   → 重复工作AI做，你有更多时间思考

2. 从"做完"到"验证"
   → 每个环节都要问：怎么证明它有效？

3. 工具会用还不够，要用透
   → 选3个工具用熟，比50个工具都试试强
```
