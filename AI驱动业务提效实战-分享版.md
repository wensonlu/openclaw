## Step 1：需求分析

**用AI做头脑风暴**

**方法：**
```
业务背景 → AI结构化拆解 → 需求优先级 → 技术评估
```

**核心：Socratic追问** — 像苏格拉底一样不断问"为什么"，挖到根因

**Prompt模板：**
```
我是一个[公司类型]的产品经理
我们现在遇到了[问题描述]
用户群体是[用户画像]
目标[业务指标提升]

请帮我分析：
1. 这个问题背后真正的原因是什么？
2. 用户有哪些未被满足的需求？
3. 有哪些可能的解决方案？
4. 你会建议先做哪个？为什么？
```

**工具：** ChatGPT / Claude / OpenClaw brainstorming

---

## Step 2：流程图+架构图

**AI帮你画图，5分钟搞定**

**核心观点：**
```
传统：手画1小时
AI：描述需求 → 5分钟出图
```

**Prompt模板（流程图）：**
```
帮我画一个[功能名称]的业务流程图
包含：主流程、异常流程、决策节点
```

**Prompt模板（架构图）：**
```
根据以下需求，帮我生成产品架构图
包含：前端模块、后端服务、数据存储、第三方服务
```

**工具：** ProcessOn AI / Whimsical AI

---

## Step 3：Prompt工程

**AI时代最重要的技能：写指令，不是写代码**

**核心公式：**
```
Prompt = 角色 + 任务 + 约束 + 成功标准
```

**示例：PRD生成**
```
你是一个资深产品经理
角色：为[产品类型]设计[功能名称]
任务：写一份完整PRD文档
约束：目标用户是[群体]，包含[功能点]
成功标准：让开发准确理解，包含用户故事和验收标准
```

**示例：用户画像**
```
你是一个用户研究专家
任务：生成3个典型用户画像
包含：用户信息、目标、痛点、使用场景
格式：表格+简短描述
```

## Step 4：原型设计 — Google Stitch

**30秒生成高保真设计稿**

**地址：** https://stitch.withgoogle.com/

**传统流程：**
```
需求 → 画原型 → 评审 → 修改 → 再评审
                  ↓
                 3-5天
```

**AI流程：**
```
需求 → AI生成UI → 评审 → 微调
                  ↓
               30分钟
```

**Prompt公式：**
```
为[产品类型]设计一个[页面类型]
包含：[元素1]、[元素2]、[元素3]
风格：[简洁/现代/专业]
目标用户：[群体]
核心目标：[用户要完成什么]
```

**示例：**
```
Design a mobile shopping app homepage for young women who love fashion.
Include: hero banner, category tabs, product grid, shopping cart icon.
Style: Modern, clean, pink and white.
Goal: Help users discover and purchase trending fashion items.
```

## Step 5：MVP开发 — Google Antigravity

**把设计稿变成可运行的版本**

**地址：** https://antigravity.google/

**核心能力：**

| 能力 | 说明 |
|------|------|
| Agent Manager | 调度多个AI并行工作 |
| Editor View | AI辅助写代码 |
| 自动测试 | 跑完告诉你对不对 |
| Artifacts | 生成截图/视频让你审核 |

**通俗解释：**
```
Agent = AI助手，但能自主行动（不只是回答问题）
Artifacts = AI的工作成果汇报（截图/视频），方便你审核
Manager View = 指挥台，可以看到多个AI同时工作
```

**工作流：**
```
描述需求 → Agent理解+规划+执行 → Artifact汇报 → 你审核 → Agent修正
```

**MVP的核心价值：**
```
不是"做完"，是"验证"
用最小成本验证：用户是否真的需要这个功能
```

## Step 6：总结

**三个核心观点：**

```
1. AI不是替代你，是放大你
   重复工作AI做，你有更多时间思考

2. 从"做完"到"验证"
   每个环节都要问：怎么证明它有效？

3. 工具会用还不够，要用透
   选3个工具用熟，比50个工具都试试强
```
