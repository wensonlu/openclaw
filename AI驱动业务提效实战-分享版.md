## Step 1：需求分析

### 是什么
**用AI做头脑风暴，把模糊的想法变成结构化的需求**

AI通过不断追问"为什么"，帮你挖到问题的根因，把散乱的想法整理成有优先级、可执行的需求文档。

### 为什么
```
传统痛点：
- 开会2小时，各说各话，没有结论
- 需求评审时各执己见，反复改
- 产品觉得说清楚了，开发理解的不一样

AI能做什么：
- 强制结构化思考（不是凭感觉）
- 追问根因，找到真正的问题
- 拆解成可评估的需求点
```

### 怎么做

**核心方法：Socratic追问**
像苏格拉底一样不断问"为什么"，直到挖到根因

**Prompt模板：**
```
我是一个[公司类型]的产品经理
我们现在遇到了[问题描述]
用户群体是[用户画像]
目标[业务指标提升]

请帮我分析：
1. 这个问题背后真正的原因是什么？
2. 用户有哪些未被满足的需求？
3. 有哪些可能的解决方案？
4. 你会建议先做哪个？为什么？
```

**实际案例：**
```
输入：
我是一个电商产品的运营
我们现在遇到了用户下单转化率低的问题
用户群体是25-35岁的职场女性
目标提升下单转化率10%

输出：
- 根因分析：支付流程复杂、优惠券使用门槛高
- 未满足需求：更简单的优惠叠加逻辑
- 解决方案：简化支付步骤、增加预估运费
- 优先建议：先做支付流程优化
```

**工具：** ChatGPT / Claude / OpenClaw brainstorming

**产出物：**
- ✅ 结构化需求文档
- ✅ 用户故事
- ✅ 优先级初步判断

## Step 2：流程图+架构图

### 是什么
**用AI自动生成业务流程图和产品架构图**

把文字描述的需求，用AI转化成可视化的流程图和架构图。

### 为什么
```
传统痛点：
- 和开发沟通不在一个频道，鸡同鸭讲
- 不知道技术边界，需求天马行空
- 不理解系统依赖，上线后踩坑

AI能做什么：
- 快速可视化想法，不需要手画
- 帮你理解"系统长什么样"
- 和开发同频，减少沟通成本
```

### 怎么做

**工具选择：**
| 工具 | 地址 | 特点 |
|------|------|------|
| ProcessOn AI | processon.com | 国产，中文界面，流程图首选 |
| Whimsical AI | whimsical.com | 英文，流程图+架构图都能做 |

**Prompt模板（流程图）：**
```
帮我画一个[功能名称]的业务流程图

包含：
- 主流程（用户主要操作路径）
- 异常流程（出错/边界情况）
- 决策节点（需要判断的地方）

用户角色：[用户/运营/管理员]
```

**Prompt模板（架构图）：**
```
根据以下需求，帮我生成产品架构图

包含：
- 前端需要哪些模块/页面
- 后端需要哪些服务
- 需要哪些数据存储
- 需要对接哪些第三方服务
```

**实际案例：**
```
输入Prompt：
帮我画一个"用户注册"的业务流程图
包含：正常注册、验证码错误、重复注册
角色：用户

输出：
主流程：输入手机号 → 获取验证码 → 输入验证码 → 设置密码 → 注册成功
异常流程1：验证码错误 → 重新获取
异常流程2：手机号已注册 → 跳转登录
决策节点：验证码是否正确 / 手机号是否已注册
```

**核心观点：**
```
传统：手画1小时
AI：描述需求，5分钟出图

产运的价值在于"想清楚"，不在于"画出来"
AI帮你把想法变成可视化图形
```

**产出物：**
- ✅ 泳道图
- ✅ 模块划分 + 服务关系图

## Step 3：Prompt工程

### 是什么
**掌握AI指令的核心公式，让AI输出你想要的结果**

Prompt不是"随便写一句话"，而是结构化的指令。好的Prompt = 角色 + 任务 + 约束 + 成功标准。

### 为什么
```
传统痛点：
- 问AI"帮我写个需求文档" → 输出笼统，没法用
- 问AI"做个好看的页面" → AI瞎猜，结果不对
- Prompt越长越好 → 反而失焦

AI能做什么：
- 结构化Prompt → 输出稳定可用
- 一次到位 → 减少反复修改
- 从"碰运气" → "可预期"
```

### 怎么做

**核心公式：**
```
Prompt = 角色 + 任务 + 约束 + 成功标准
```

**公式拆解：**
```
角色：AI是什么身份？（资深产品经理？用户研究员？）
任务：要做什么？（写PRD？生成用户画像？）
约束：有什么限制？（字数/风格/格式）
成功标准：什么样的输出算成功？（开发能看懂/包含验收标准）
```

**示例1：PRD生成**
```
角色：你是一个资深产品经理
任务：帮我写一份完整的PRD文档
约束：
  - 目标用户是[25-35岁职场女性]
  - 需要包含[核心功能、用户路径、异常流程]
  - 限制条件是[2周内上线，技术资源有限]
  - 文档字数控制在2000字以内
成功标准：
  - 能够让开发团队准确理解需求
  - 包含用户故事、验收标准、非功能需求
```

**示例2：用户画像**
```
角色：你是一个用户研究专家
任务：为[新功能]生成3个典型用户画像

约束：
  - 每个画像包含：基本信息、目标、痛点、场景
  - 格式：表格+简短描述
  - 不超过500字

成功标准：
  - 能让团队快速理解目标用户
  - 能指导功能设计决策
```

**示例3：竞品分析**
```
角色：你是一个竞品分析师
任务：分析[竞品名称]的[功能点]

约束：
  - 从核心功能、用户体验、商业模式、技术架构分析
  - 格式：对比表格 + 关键洞察
  - 重点突出优劣势

成功标准：
  - 能指导我们自己的产品决策
  - 能发现竞品没做好的机会点
```

**工具：** ChatGPT / Claude

**产出物：**
- ✅ 可执行的AI指令，直接用于下一步

## Step 4：原型设计 — Google Stitch

### 是什么
**用AI把需求描述变成高保真设计稿**

Google Stitch：输入自然语言描述，AI直接生成UI界面。30秒出初稿，快速验证"这个想法长什么样"。

**地址：** https://stitch.withgoogle.com/

### 为什么
```
传统痛点：
- 等设计师排期：1-3天
- 原型改了三版：1周过去了
- 想法没法快速验证，只能靠脑补

AI能做什么：
- 30秒出初稿，不用等设计师
- 快速验证想法是否可行
- 产运主导，不需要依赖设计资源
```

### 怎么做

**Prompt公式：**
```
为[产品类型]设计一个[页面类型]

包含以下元素：
1. [元素1，如导航栏]
2. [元素2，如用户头像]
3. [元素3，如内容列表]

设计风格：[简洁/现代/专业/活泼]
目标用户：[用户群体]
核心目标：[用户进来要完成什么任务]
```

**示例1：电商APP首页**
```
Design a mobile shopping app homepage for young women who love fashion.
Include: hero banner, category tabs, product grid, shopping cart icon, user profile.
Style: Modern, clean, pink and white color scheme.
Goal: Help users discover and purchase trending fashion items quickly.
```

**示例2：内部审批界面**
```
Design an internal approval workflow interface for managers.
Include: pending approval list, approval/reject buttons, detail panel, history timeline.
Style: Professional, enterprise, blue color scheme.
Goal: Let managers quickly review and make decisions on team requests.
```

**示例3：数据看板**
```
Design a data dashboard for operations team to monitor daily KPIs.
Include: date range selector, KPI cards, trend charts, export button, alert panel.
Style: Data-focused, clean, dark mode optional.
Goal: Help ops team quickly identify anomalies and take action.
```

**传统 vs AI流程对比：**
```
传统流程：
需求 → 画原型 → 评审 → 修改 → 再评审
                  ↓
                 3-5天

AI流程：
需求 → AI生成UI → 评审 → 微调
                  ↓
               30分钟
```

**关键点：**
- Prompt越具体，结果越准确
- 不是"一次成型"，是"快速出稿"
- 产运的价值在于"想清楚要什么"，不在于"画出来"

**产出物：**
- ✅ 高保真设计稿（分钟级）

## Step 5：MVP开发 — Google Antigravity

### 是什么
**用AI Agent把设计稿变成可运行的代码**

Google Antigravity：Agent可以自主理解需求、写代码、测试、生成汇报。你只需要描述需求，AI帮你完成剩下的工作。

**地址：** https://antigravity.google/

### 为什么
```
传统痛点：
- 等开发排期：2-4周
- 开发说"做不了"：沟通成本高
- 想法验证周期太长

AI能做什么：
- 30分钟出可运行版本
- Agent自主完成，不需要你盯着
- 用Artifact（截图/视频）汇报，你来审核
```

### 怎么做

**Antigravity四大能力：**

| 能力 | 说明 |
|------|------|
| Agent Manager | 调度多个AI Agent并行工作，同时做多件事 |
| Editor View | 传统AI辅助编程，你参与时用这个 |
| 自动测试 | AI跑完测试，告诉你对不对 |
| Artifacts | AI的工作成果汇报（截图/视频），方便你审核 |

**核心概念解释：**

| 概念 | 通俗解释 |
|------|---------|
| Agent | AI助手，但能自主行动（不只是回答问题） |
| Manager View | 指挥台，可以看到多个AI同时工作 |
| Artifacts | AI的工作成果汇报（截图/视频），方便你审核 |
| Editor View | 传统编程界面，AI在旁边辅助你写 |

**工作流：**
```
Step 1: 描述需求
        ↓
Step 2: Agent理解 → 规划 → 执行
        ↓
Step 3: 生成Artifact汇报进展
        ↓
Step 4: 你审核，给反馈
        ↓
Step 5: Agent修正，继续执行
```

**MVP场景示例：**
```
需求描述：
创建一个用户注册页面，包含：
- 邮箱/手机号输入
- 密码设置（需包含大小写+数字）
- 用户协议勾选
- 注册按钮

注册成功后跳转到欢迎页

要求：
- 响应式设计（手机/电脑都正常）
- 输入实时校验
- 密码强度提示
```

**MVP的核心价值：**
```
不是"做完"，是"验证"
用最小成本验证：用户是否真的需要这个功能

案例：
如果你的假设是"用户愿意付费"
MVP只需要：展示产品+付费按钮
不需要：完整支付流程、发票系统、客服体系
```

**产出物：**
- ✅ 可运行的MVP版本

## Step 6：总结

### 今天回去可以做的3件事

```
□ 用Stitch生成一个你手头的需求原型
□ 用ChatGPT写一个你常用的Prompt模板
□ 想想你工作中最费时间的3件事，有没有AI方案
```

### 三个核心观点

```
1. AI不是替代你，是放大你
   重复工作AI做，你有更多时间思考

2. 从"做完"到"验证"
   每个环节都要问：怎么证明它有效？

3. 工具会用还不够，要用透
   选3个工具用熟，比50个工具都试试强
```
