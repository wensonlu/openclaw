### 【第4页：Step 1 - 需求分析】

**标题：** Step 1：需求分析

**核心方法：**
```
业务背景 → AI结构化拆解 → 需求优先级 → 技术评估
```

**左侧：方法说明**
```
AI做的是什么？
- Socratic追问：不断问"为什么"，挖到根因
- 结构化拆解：把模糊的想法变成具体的需求点
- 优先级判断：帮你理清先做哪个
```

**右侧：演示Prompt模板**
```
我是一个[公司类型]的产品经理
我们现在遇到了[问题描述]
用户群体是[用户画像]
目标[业务指标提升]

请帮我分析：
1. 这个问题背后真正的原因是什么？
2. 用户有哪些未被满足的需求？
3. 有哪些可能的解决方案？
4. 你会建议先做哪个？为什么？
```

**讲解要点（8分钟）：**
```
（3分钟）讲解Socratic追问方法
像苏格拉底一样不断追问"为什么"

（5分钟）现场演示
输入：运营同学的一个痛点
输出：结构化的需求文档

关键：让观众看到"AI是怎么追问的"
```

**标题：** Step 2：AI帮你画流程图和架构图

**核心观点（大大字）：**
```
传统：手画1小时

**流程图**

**架构图**

"根据需求生成产品架构图，

```

**讲解要点（5分钟）：**
```
（1分钟）讲核心观点
不需要手画，描述需求AI自动生成

（2分钟）演示ProcessOn AI
打开 → 输入Prompt → 5秒生成 → 展示结果

（2分钟）关键点
产运的价值在于"想清楚"，不在于"画出来"
AI帮你把想法可视化，和开发同频沟通
```

### 【第6页：Step 3 - Prompt工程】

**标题：** Step 3：Prompt工程

**核心公式（大字居中）：**
```
┌─────────────────────────────────────────┐
│                                         │
│   Prompt = 角色 + 任务 + 约束 + 成功标准  │
│                                         │
└─────────────────────────────────────────┘
```

**下方：两个示例场景（卡片形式）**
```
┌──────────┐  ┌──────────┐
│ PRD生成  │  │ 用户画像  │
└──────────┘  └──────────┘
```

**讲解要点（8分钟）：**
```
（3分钟）讲公式
用做菜比喻：
- 角色：你是什么厨师
- 任务：你要做什么菜
- 约束：有什么限制
- 成功标准：做出来是什么味道

（5分钟）现场演示
选一个场景，观众投票
粘贴Prompt → 展示输出

关键：同样的工具，Prompt不同效果天差地别
```

### 【第8页：Step 4 - 原型设计】

**标题：** Step 4：原型设计

**上方：传统 vs AI对比**
```
传统流程：需求 → 画原型 → 评审 → 修改 → 再评审
                    ↓
                   3-5天

AI流程：需求 → AI生成UI → 评审 → 微调
                    ↓
               30分钟出初稿
```

**下方：Stitch核心能力**
```
• 输入自然语言 → 输出UI设计
• 支持复杂界面描述
• Google设计语言
• 快速验证"这个想法长什么样"
```

**Prompt模板：**
```
Design a [产品类型] [页面类型].

Include:
1. [元素1]
2. [元素2]
3. [元素3]

Style: [简洁/现代/专业]
Target users: [用户群体]
Core goal: [用户目标]
```

**讲解要点（7分钟）：**
```
（2分钟）讲清价值
不是替代设计师，是"快速验证想法"
30分钟 vs 3天，差距在哪里

（5分钟）讲Prompt模板
套用公式，现场生成一个案例

重点：
产运的价值在于"想清楚要什么"，不在于"画出来"
```

### 【第9页：Step 5 - MVP开发】

**标题：** Step 5：MVP开发

**工作流（横向流程图）：**
```
设计稿描述 → Antigravity理解 → 代码生成 → 自动测试 → Artifact汇报
                                ↓
                            你来审核
```

**Antigravity四大能力：**
```
Agent Manager → 调度多个AI并行工作
Editor View → AI辅助写代码
Artifacts → 截图/视频汇报
自动测试 → 跑完告诉你对不对
```

**MVP的核心价值：**
```
不是"做完"，是"验证"
用最小成本验证：用户是否真的需要这个功能
```

**讲解要点（12分钟）：**
```
（3分钟）介绍Antigravity
Google出品，Agent自主完成复杂任务

（4分钟）讲MVP逻辑
重点：不是"做一个完整产品"
是"做一个最小版本，验证核心假设"

（3分钟）讲Artifacts
你审核截图/视频，不用盯屏幕

（2分钟）演示Agent执行过程
描述需求 → 展示Agent工作 → 展示Artifact输出
```
